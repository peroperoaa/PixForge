# 📄 游戏像素资产自动化工作流方案摘要

## 一、 工作流总体架构图

```text
[用户输入] ──(文本要求 / 非像素原画草图)
   │
   ▼
【模块一：提示词生成与视觉解析】(云端 API) 
   │   └─ 核心：多模态大语言模型（MLLM）反推或扩写优质提示词
   ▼
【模块二：高清原画生成】(云端 API) 
   │   └─ 核心：文生图大模型根据提示词，生成非像素风格的高清概念原图
   ▼
【模块三：ComfyUI 像素化引擎】(本地显卡运行)
   │   └─ 核心：接收高清原图，通过 ControlNet 与像素化大模型进行图生图重绘
   ▼
【模块四：Python 降维与处理】(本地脚本运行)
       └─ 核心：执行严格的算法级缩放、色彩量化与背景去除，导出最终资产
```

---

## 二、 核心模块技术选型与执行细节

### 模块一：提示词生成与视觉解析 (Prompt Generation)
*   **定位**：工作流的“大脑”，负责理解用户意图、提取上传图像特征，并输出结构化的优质提示词（Prompt）。
*   **运行环境**：云端 API 调用。
*   **厂商与模型选择池**(暂定gemini，后续拓展)：
        *   **Google**: `gemini-3.1-pro-preview`, `gemini-3-flash-preview`

### 模块二：高清原画生成 (Original Concept Generation)
*   **定位**：工作流的“主画师”，负责将文字概念转化为高质量、结构清晰的非像素概念图，为后续像素化提供完美的“骨架”。
*   **运行环境**：云端 API 调用。
*   **厂商与模型选择池**(暂定gemini，后续拓展)：
        *   **Google**: `gemini-3-pro-image-preview`, `gemini-3.1-flash-image-preview`

### 模块三：ComfyUI 像素化转换引擎 (Pixelization Engine)
*   **定位**：工作流的“风格化滤镜”，将云端生成的高清原图转换为具备像素排布规律（网格感）的高分辨率伪像素图。
*   **运行环境**：**本地显卡运行**，通过开启 `--listen` 暴露本地 API 供工作流调用。
*   **工作流节点设计 (ComfyUI Workflow)**：
    1.  **输入节点**：加载“模块二”传入的高清原图。
    2.  **线稿提取 (ControlNet)**：使用 `ControlNet Lineart` 或 `Canny` 提取图像边缘，确保转换时不丢失剑刃、眼睛等核心轮廓。
    3.  **大模型重绘 (Img2Img)**：
        *   加载专用的像素画底模（如 *Pixel Art XL* 或 *Retro Game Assets Checkpoint*）。
        *   设置较低的重绘幅度（Denoising 0.45 ~ 0.6），在保持原图结构的基础上赋予像素块质感。
    4.  **输出节点**：输出一张 512x512 或 1024x1024 带有马赛克质感的图像。

### 模块四：Python 核心处理与资产导出 (Post-processing)
*   **定位**：工作流的“精雕师”，用数学算法解决 AI 无法精确控制物理像素点和色彩数量的痛点。
*   **运行环境**：**本地运行**（Python 环境，由主工作流脚本直接执行）。
*   **核心处理管线**：
    1.  **AI 抠图 (Background Removal)**：调用本地 `rembg` 库（基于 U-2-Net），精准剔除背景，生成带有 Alpha 透明通道的图像。
    2.  **色彩量化 (Color Quantization)**：利用 `scikit-learn` 的 K-Means 聚类算法，将复杂的 AI 颜色强制压缩限制在指定的色阶内（如：8色、16色、GB风格4色，或载入指定的 Lospec 调色板），彻底消除“AI杂色”。
    3.  **算法降维 (Nearest Neighbor Downscaling)**：使用 `PIL/OpenCV` 的**最近邻插值算法 (Nearest Neighbor)**，将图像严格等比缩小至目标游戏尺寸（如 32x32, 64x64）。该算法保证边缘硬朗，绝不产生模糊像素。
    4.  **资产保存**：导出为 `[名称]_32x32.png` 格式，直接可拖入 Unity/Godot 引擎使用。
